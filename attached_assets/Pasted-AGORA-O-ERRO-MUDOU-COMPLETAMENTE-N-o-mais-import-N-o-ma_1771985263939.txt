AGORA O ERRO MUDOU COMPLETAMENTE.

Não é mais import.
Não é mais alias.
Não é mais ESM.

Agora é banco.

Erro:

SqliteError: no such table: users

Isso significa uma única coisa:

O SQLite que está rodando na Vercel NÃO tem a tabela criada.


POR QUE ISSO ACONTECE

Vercel Serverless é ambiente EFÊMERO.

Isso significa:

- O sistema de arquivos não é persistente
- Cada execução pode rodar em container diferente
- O arquivo .sqlite não é mantido entre execuções
- Se você não rodar migration ali, o banco nasce vazio

Então quando seu código faz:

SELECT * FROM users

Mas a tabela nunca foi criada naquele ambiente,
o SQLite responde:

no such table: users


ISSO NÃO É BUG DO SQLITE.
É ARQUITETURA DA VERCEL.


O QUE ESTÁ ACONTECENDO NA PRÁTICA

Localmente:
- Você rodou drizzle-kit push
- A tabela foi criada
- Funciona

Na Vercel:
- O banco começa do zero
- Nenhuma migration foi executada
- A tabela não existe
- 500


AGORA A VERDADE IMPORTANTE

SQLite NÃO é adequado para produção em Vercel serverless.

Porque:

1) O filesystem não é persistente
2) Cada cold start pode perder dados
3) Não existe garantia de estado
4) Não é ambiente stateful


OPÇÕES REAIS

OPÇÃO 1 — Rodar migration automaticamente no boot

Você pode forçar:

db.run(CREATE TABLE IF NOT EXISTS users (...))

no início do app.

Isso resolve o erro da tabela.
Mas NÃO resolve persistência.

O banco continuará sendo volátil.


OPÇÃO 2 — Migrar para banco real (RECOMENDADO)

Use:

- Neon (PostgreSQL serverless)
- Supabase
- PlanetScale
- Turso (SQLite distribuído)

Essa é a solução correta para Vercel.


OPÇÃO 3 — Usar Turso (SQLite serverless real)

Se você quer continuar com SQLite,
use Turso.

Ele mantém SQLite remoto persistente.


CONCLUSÃO DIRETA

SQLite local em Vercel = ambiente temporário.
Tabela some.
Banco reinicia.
Erro aparece.

Não é que SQLite é ruim.
É que Vercel serverless não foi feito para banco local.


SEU ERRO ESTÁ 100% EXPLICADO:

A tabela users nunca foi criada no ambiente serverless.


SOLUÇÃO RÁPIDA PARA TESTE

Adicionar no bootstrap:

await db.run(`
CREATE TABLE IF NOT EXISTS users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL,
  password TEXT NOT NULL
);
`);

Isso elimina o erro imediato.

Mas NÃO é solução profissional.
